### `crates/base60-cli/src/lib.rs` — `__bench` module addition (Claude's Discretion per CONTEXT §J; recommended per RESEARCH ARCH RESP MAP)

**Analog:** identical to `__fuzz` above — the same `#[doc(hidden)] pub mod __name { pub use crate::…::…; }` idiom — minus the `#[cfg(fuzzing)]` gate (benches compile on every toolchain).

**Excerpt to adapt:**
```rust
/// Hidden re-exports for this crate's `benches/` binaries.
///
/// Criterion benches compile as external binaries that link only the
/// library's `pub` surface. The functions exercised here (`dump_all`,
/// `decode_stream`, `find_all`, `PALETTE_NONE`, `InputFormat`) are
/// `pub(crate)` by design; re-exporting them under `__bench` keeps the
/// non-bench public API unchanged. Unstable-by-definition; see the
/// crate-level note on `__TuiTimeScale`.
#[doc(hidden)]
pub mod __bench {
    pub use crate::cli::InputFormat;
    pub use crate::color::PALETTE_NONE;
    pub use crate::decode::decode_stream;
    pub use crate::dump::dump_all;
    pub use crate::search::find_all;
}
```

**Key convention notes:**
- `pub use` inside a `#[doc(hidden)]` module is NOT visible to `cargo doc --workspace --no-deps --locked` (Phase 3 D-06 precedent: `__TuiTimeScale` does not appear in generated docs).
- No new `pub(crate) → pub` widening required on the underlying items — `pub use crate::X::Y` inside a `pub mod` re-exports `Y` with the containing module's visibility (Rust RFC: re-exports widen visibility of the alias, not the source).

**Source:** `crates/base60-cli/src/lib.rs` lines 37-45 (`__TuiTimeScale` + `__test_hooks`).

**Apply to:**
- `__fuzz` module in `lib.rs` (D-05, gated by `#[cfg(fuzzing)]`).
- `__bench` module in `lib.rs` (RESEARCH ARCH RESP MAP Option A; recommended over Option B `pub(crate) → pub` scatter widening).

**Canonical excerpt:**
```rust
#[doc(hidden)]
pub use cli::TimeScale as __TuiTimeScale;
```

**Why this works:** `pub use crate::X::Y` inside a `#[doc(hidden)] pub mod` widens the alias's visibility to `pub` while `cargo doc --workspace --no-deps --locked` skips it. Non-fuzz / non-bench consumers see no change in `base60::` public API. Phase 3 D-06 already vetted this shape against `cargo doc` + `RUSTDOCFLAGS=-D warnings`.

### Shared Pattern 5: Deterministic compile-time bench input

**Source:** RESEARCH.md Example 4 (lines 726-735). No in-repo analog — Phase 5 introduces this. In-repo adjacent idiom: `crates/base60-cli/src/chunk.rs:27` (`pub(crate) const fn be_u64(…)`) shows the `const fn` family the workspace already uses.

**Apply to:** every bench file requiring more than a handful of input values.

**Two canonical forms:**

Form A — `u64` input array:
```rust
const FIB_MUL: u64 = 0x9E37_79B9_7F4A_7C15;
const INPUTS: [u64; 1024] = {
    let mut arr = [0_u64; 1024];
    let mut i = 0_u64;
    while i < 1024 {
        arr[i as usize] = i.wrapping_mul(FIB_MUL);
        i += 1;
    }
    arr
};
```

Form B — `u8` haystack:
```rust
const SIZE: usize = 1 << 20;
const INPUT: [u8; SIZE] = {
    let mut arr = [0_u8; SIZE];
    let mut i: usize = 0;
    while i < SIZE {
        arr[i] = (i.wrapping_mul(13).wrapping_add(7)) as u8;
        i += 1;
    }
    arr
};
```

**Key convention:** `std::hint::black_box` on BOTH the input reference and any accumulator prevents the optimiser from constant-folding the loop body across `b.iter` iterations. All bench examples follow this.

## Key Conventions for Planner

1. **Visibility shims beat visibility scatter.** `__fuzz` and `__bench` modules in `lib.rs` are symmetric, both use `#[doc(hidden)] pub mod …`, and keep `pub(crate)` items (`parse_run`, `RUN_LEN`, `dump_all`, `decode_stream`, `find_all`, `PALETTE_NONE`, `InputFormat`) unchanged at their definition sites. The only non-shim visibility bump is on `parse_run` (fn) and `RUN_LEN` (const) per D-06 — these need to go `pub(crate)` because `pub use crate::decode::{parse_run, RUN_LEN}` requires the items to already be at least `pub(crate)`-visible.

2. **Workspace lints apply to benches but NOT to `fuzz/`.** Benches live inside `crates/base60-cli/` and `crates/base60-core/` so they inherit `workspace.lints` — expect `clippy::pedantic` + `nursery` + `cargo` warnings. Fuzz crate has its own `[workspace]` and is the opposite: no workspace lints inherited (intentional per CONTEXT §Established Patterns line 275).

3. **`#[cfg(test)] mod tests` inline style STILL applies to `lib.rs`.** The optional `fuzz_module_absent` compile-time assertion (RESEARCH lines 447-458) goes into the existing `#[cfg(test)] mod tests` block at `lib.rs:195-249`, NOT a new test file.

4. **Alphabetical order within `[dev-dependencies]`.** Existing `base60-cli/Cargo.toml` lists them alphabetically; `criterion` slots between `base60-core` and `predicates`. `base60-core/Cargo.toml` currently has only `serial_test` — `criterion` comes before it alphabetically, so swap the order.
